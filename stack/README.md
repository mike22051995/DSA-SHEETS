## 🎯 Key Stack Patterns You MUST Master

### 1. **Monotonic Stack** ⭐⭐⭐
Most important pattern for FAANG! Used in 40% of stack problems.
- **Problems**: 4, 11, 14, 15, 21, 27, 28
- **Key Concept**: Maintain elements in increasing or decreasing order
- **Use Cases**: Next greater/smaller element, area calculations, optimization problems

### 2. **Stack for Matching/Pairing** ⭐⭐⭐
Essential for parentheses and bracket problems.
- **Problems**: 2, 6, 7, 9, 19, 25
- **Key Concept**: Push opening elements, pop when matching closing elements
- **Use Cases**: Valid parentheses, duplicate removal, string matching

### 3. **Stack for Expression Evaluation** ⭐⭐⭐
Calculator problems are VERY common in FAANG interviews!
- **Problems**: 16, 23, 24, 30
- **Key Concept**: Handle operator precedence and parentheses
- **Use Cases**: Mathematical expression parsing and evaluation

### 4. **Two Stack Technique** ⭐⭐
Creative use of multiple stacks.
- **Problems**: 3, 5, 18
- **Key Concept**: Use auxiliary stack for additional functionality
- **Use Cases**: Min/Max tracking, queue implementation, validation

### 5. **Stack for Nested Structures** ⭐⭐
Handle nested data like encoded strings or nested parentheses.
- **Problems**: 13, 17, 19
- **Key Concept**: Push state when going deeper, pop when completing level
- **Use Cases**: Decode strings, path simplification, scoring

### 6. **Stack Simulation** ⭐⭐
Simulate stack operations or validate sequences.
- **Problems**: 8, 10, 18
- **Key Concept**: Track operations and validate constraints
- **Use Cases**: Game scoring, navigation, sequence validation

### 7. **Stack with HashMap** ⭐
Combine stack with hash table for efficient lookups.
- **Problems**: 4, 29
- **Key Concept**: Stack for order, HashMap for O(1) access
- **Use Cases**: Frequency tracking, element mapping

### 8. **Advanced Stack (Histogram/Area)** ⭐⭐⭐
Most challenging and frequently asked in FAANG!
- **Problems**: 21, 22, 26, 28
- **Key Concept**: Calculate areas using stack-based boundary detection
- **Use Cases**: Largest rectangle, water trapping, maximal rectangle

## 🚀 Quick Reference Guide

### When to Use Stack:
- ✅ Need to match pairs (parentheses, brackets)
- ✅ Finding next greater/smaller elements
- ✅ Evaluating expressions with precedence
- ✅ Processing nested structures
- ✅ Need LIFO (Last In First Out) behavior
- ✅ Calculating areas/spans with boundaries

### Stack Implementation Tips:
```cpp
// C++ STL Stack
stack<int> st;
st.push(x);      // Add element
st.pop();        // Remove top
st.top();        // Access top
st.empty();      // Check if empty
st.size();       // Get size

// Common patterns:
while (!st.empty() && condition) {
    // Process and pop
    st.pop();
}
```

### Time Complexity Patterns:
- Most stack problems: **O(n)** time, **O(n)** space
- Each element pushed and popped at most once
- Amortized O(1) per operation
