## 🔍 Pattern Recognition Guide

### When you see these keywords:

**"Two elements sum to..."** → Hash Map or Two Pointers  
**"Contiguous subarray"** → Sliding Window or Kadane's  
**"Without repeating"** → Sliding Window with Set  
**"Longest/Shortest substring"** → Sliding Window  
**"All permutations/combinations"** → Backtracking  
**"Number of ways"** → Dynamic Programming  
**"Minimum/Maximum path"** → DP or DFS/BFS  
**"Top K elements"** → Heap  
**"Intervals/Meetings"** → Sorting + Greedy/Heap  
**"Rotated sorted array"** → Modified Binary Search  
**"Parentheses matching"** → Stack  
**"Next greater/smaller"** → Monotonic Stack  
**"Level by level"** → BFS  
**"All paths"** → DFS/Backtracking  
**"Connected components"** → Union Find or DFS  
**"Shortest path"** → BFS or Dijkstra  
**"Detect cycle"** → Fast-Slow Pointers or DFS

## 💡 Interview Tips

### Before Interview:
- Review pattern recognition guide
- Practice on whiteboard/paper
- Time yourself (45 minutes per problem)
- Practice explaining your thought process

### During Interview:
1. **Clarify requirements** (5 min)
   - Input format, constraints
   - Edge cases
   - Expected output

2. **Discuss approach** (10 min)
   - Start with brute force
   - Optimize step by step
   - Explain time/space complexity

3. **Code** (20 min)
   - Write clean, readable code
   - Use meaningful variable names
   - Add comments for complex logic

4. **Test** (10 min)
   - Walk through with examples
   - Check edge cases
   - Discuss potential bugs

## ⚡ Quick Reference

### Time Complexities:
- **O(1)**: Hash table lookup, array access
- **O(log n)**: Binary search, balanced tree operations
- **O(n)**: Linear scan, two pointers
- **O(n log n)**: Sorting, heap operations
- **O(n²)**: Nested loops, DP 2D
- **O(2^n)**: Backtracking, recursion without memo

### Space Complexities:
- **O(1)**: Constant space
- **O(log n)**: Recursion stack for balanced tree
- **O(n)**: Hash table, DP array, recursion for linked list
- **O(n²)**: 2D DP matrix
